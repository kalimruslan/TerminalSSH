# TerminalSSH

**Мобильный SSH-клиент для Android** — современное приложение для удалённого управления серверами с телефона.

## Зачем этот проект?

TerminalSSH создан для системных администраторов, DevOps-инженеров и разработчиков, которым нужен быстрый доступ к серверам в любое время и в любом месте. Приложение позволяет:

- Подключаться к удалённым серверам по SSH-2
- Выполнять команды в интерактивном терминале
- Сохранять часто используемые подключения
- Добавлять команды в избранное для быстрого доступа

## Почему это может быть интересно?

### Для пользователей
- Полноценный SSH-терминал в кармане
- Сохранение подключений с быстрым доступом
- Избранные команды, привязанные к конкретным серверам
- Современный Material Design 3 интерфейс

### Для разработчиков
Проект демонстрирует профессиональные подходы к Android-разработке:

- **Clean Architecture** — чёткое разделение на Domain, Data и Presentation слои
- **MVI паттерн** — предсказуемое управление состоянием (State, Intent, Effect)
- **Модульная структура** — 8 независимых модулей для масштабируемости
- **Jetpack Compose** — современный декларативный UI
- **Kotlin Coroutines + Flow** — реактивная асинхронность
- **Hilt** — dependency injection

## Основные функции

### SSH-подключение
- Подключение по хосту, порту, логину и паролю
- Автоматическое создание shell-сессии с PTY
- Обработка ANSI escape-последовательностей

### Терминал
- Выполнение команд в реальном времени
- Цветовое выделение (команды — жёлтый, вывод — белый, ошибки — красный)
- Автоскролл к последней строке
- Буферизация вывода для оптимизации производительности

### Управление подключениями
- Сохранение подключений с именем
- Быстрый выбор из списка сохранённых
- Сортировка по времени последнего использования

### Избранные команды
- Добавление команд в избранное одним нажатием
- Привязка команд к конкретным подключениям
- Быстрый доступ через выпадающее меню

## Архитектура

### Clean Architecture

```
┌─────────────────────────────────────────────────┐
│                 Presentation                     │
│  (Compose UI, ViewModels, MVI State/Intent)     │
└─────────────────────┬───────────────────────────┘
                      │
┌─────────────────────▼───────────────────────────┐
│                   Domain                         │
│  (Use Cases, Repository Interfaces, Models)     │
└─────────────────────┬───────────────────────────┘
                      │
┌─────────────────────▼───────────────────────────┐
│                    Data                          │
│  (Repository Impl, SSH Client, Room Database)   │
└─────────────────────────────────────────────────┘
```

### MVI паттерн

- **State** — неизменяемое состояние экрана
- **Intent** — пользовательские действия
- **Effect** — одноразовые события (навигация, scroll)
